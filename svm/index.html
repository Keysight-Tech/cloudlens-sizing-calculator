The CloudLens sVM (service VM) calculator is currently being built.

sVM is a fundamentally different deployment model from the VM Sensor or K8s DaemonSet — it runs one service VM per ESXi host, taps traffic via vSphere Distributed Switch / NSX port-mirror or PVLAN, and delivers it to the CloudLens Manager. Sizing depends on:

  • Per-host throughput capacity — NOT per-VM, since one sVM serves all guests on an ESXi host
  • vSphere version + topology — vSphere 7/8, standard vs distributed switch, NSX-T overlay
  • Mirror source method — DVMirror, ERSPAN, port-mirror group, PVLAN
  • sVM footprint budget — typically 2–8 vCPU / 4–16 GB per ESXi host
  • Guest VM density — high-density hosts (100+ VMs) need larger sVMs
